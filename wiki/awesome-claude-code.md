# Awesome Claude Code — Catálogo

> Lista curada de skills, agentes, plugins, hooks y herramientas para Claude Code. Mantenida por hesreallyhim, con PRs abiertos solo para Claude.

## Contexto

El repo `hesreallyhim/awesome-claude-code` es la referencia canónica de la comunidad Claude Code. Tiene ~150 recursos en 7 categorías. El criterio de curaduría es alto — no es un dump de repos sino una selección de calidad.

## Detalle

### Las 7 categorías

| Categoría | Qué contiene |
|-----------|-------------|
| **Agent Skills** | Colecciones de skills para tareas especializadas |
| **Workflows & Knowledge Guides** | Frameworks y metodologías de trabajo |
| **Tooling** | Apps construidas sobre Claude Code |
| **Status Lines** | Customizaciones de la barra de estado |
| **Hooks** | Librerías y herramientas para hooks |
| **Slash Commands** | Comandos por categoría (git, testing, docs, etc.) |
| **CLAUDE.md Files** | Ejemplos reales de proyectos de producción |

### Highlights por categoría

**Agent Skills** — los más valorados:
- `Trail of Bits Security Skills` — 21 skills de seguridad profesional (CodeQL, Semgrep, variant analysis)
- `Claude Scientific Skills` (K-Dense) — ciencia, ingeniería, finanzas, escritura. Nivel PhD
- `cc-devops-skills` — IaC para cualquier plataforma, validaciones, generadores
- `Everything Claude Code` — cubre casi todos los features de Claude Code con valor standalone
- `Context Engineering Kit` — técnicas avanzadas, mínimo footprint de tokens

**Workflows** — los más importantes:
- [[ralph-wiggum-technique]] — loop autónomo hasta completar task. El patrón emergente más discutido
- `Agentic Workflow Patterns` — diagramas Mermaid de todos los patrones de Anthropic
- `AB Method` — spec-driven workflow con subagentes especializados por fase del SDLC
- `RIPER Workflow` — 5 fases: Research, Innovate, Plan, Execute, Review

### Cómo contribuir

Solo se puede recomendar recursos via issues (template automático). PRs solo los abre Claude. No es un directorio de cualquier cosa — cada recurso tiene una descripción evaluativa.

### Señal de calidad

El autor usa lenguaje valorativo en las descripciones: "really one of the best skills repos", "worth downloading even just as documentation", "I think it's incredibly promising". Eso ayuda a priorizar qué explorar primero.
